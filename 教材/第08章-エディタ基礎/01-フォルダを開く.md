# フォルダを開く（Cursor/VS Code）

## たとえ話

> 何かを作る作業に取りかかるとき、慣れた人は道具と材料を一度、作業台の上にまとめて広げる。引き出しを開け閉めしながら一つずつ取り出すより、目の前に全部並んでいるほうが、手が止まらずに進むからだ。広げるという最初のひと手間が、その後の流れを軽くする。

> パソコンでの作業も、これと同じだ。ファイルを一つずつ開く代わりに、関係する資料の入ったフォルダをまるごとエディタに広げると、見ながら直す・探すが一つの画面で進む。今日学ぶのは、その「作業台にフォルダを広げる」操作だ。黒い画面に見えても、中身はFinderの強化版にすぎない。

## なぜ大事か

Finderでもファイルは開けます。エディタでフォルダを開くと、**関連ファイルを一覧で見ながら編集・検索** できます。Rebuild AI Guild の教材作成や、相談セットの編集にもつながります。

例：`01_サービス・料金` の中のファイルを一覧表示し、クリックして内容をプレビューする、という具合です。

## わからないまま進まないチェック

- **仕事フォルダがない** → 第6章テーマ3に戻る。書類に仮フォルダ `仕事` を作ってもOK
- **フォルダを開くと何も出ない** → 空フォルダかもしれない。第6章で作ったフォルダを選ぶ

## 読んで学ぶ

**ワークスペース** とは、エディタで今開いている作業フォルダのことです。イメージは「作業机の上に広げたフォルダ一式」です。

今日触るのは **ファイルメニュー** と **左サイドバー** だけです。AIチャットパネルは開かなくて大丈夫です。

**個人情報・機密情報の注意**：スクショにお客さまの実名ファイルが写らないよう確認してください。

## 手順

### Cursor の場合

1. Dockまたはアプリケーションから **Cursor** を開く
2. 画面上部メニュー **ファイル** → **フォルダを開く…**（英語UIなら File → Open Folder…）
3. ダイアログで **書類** → **仕事** フォルダを選ぶ
4. **開く** をクリック
5. 左サイドバーにフォルダツリー（`01_サービス・料金` など）が表示されることを確認
6. ウィンドウ上部にフォルダ名が表示されていることを確認

**スクショを撮るなら**：「フォルダを開く」ダイアログ、開いた後のサイドバー

### VS Code の場合

1. **Visual Studio Code** を開く
2. **ファイル** → **フォルダーを開く…**
3. 以降は Cursor と同様（書類 → 仕事 → 開く）

### 確認作業（5分）

1. サイドバーで `01_サービス・料金` の左の **三角** をクリックして展開
2. 中のファイルを1つクリック → 右側に内容が表示されればOK（空なら第6章に戻ってファイルを入れる）

## できたらOK

- エディタで `仕事` フォルダが開いている
- 左サイドバーにサブフォルダが見える
- ファイルを1つクリックして内容を表示できた
- AIチャットは使っていない（使ってもOKですが、今日の必須ではありません）
